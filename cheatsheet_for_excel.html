# 명령어 ('~해줘') 의미 실제 진행 사항
1. 파일 보기 / 읽기
1-1"이 파일 읽어줘" / "파일 내용 보여줘"특정 파일의 코드/텍스트를 화면에 출력**Read Tool** 실행 → 파일 시스템(File System)에서 해당 경로의 파일을 읽어 텍스트로 반환
1-2"프로젝트 구조 보여줘" / "폴더 구조 알려줘"프로젝트의 디렉토리/파일 트리 확인**Bash(`ls`, `tree`)** 또는 **Glob Tool** 실행 → 디렉토리 트리(Directory Tree)를 탐색하여 계층 구조 출력
1-3"이 함수 찾아줘" / "어디서 쓰이는지 찾아줘"코드 내 특정 키워드/함수 검색**Grep Tool** 실행 → 정규표현식(Regex) 기반 전문 검색(Full-text Search)으로 파일 내용을 탐색
1-4"*.js 파일 다 찾아줘"특정 확장자/패턴의 파일 목록 조회**Glob Tool** 실행 → 글로브 패턴(Glob Pattern, 예: `**/*.js`)으로 파일 시스템을 매칭 탐색
2. 파일 수정 / 생성
2-1"이 부분 수정해줘" / "바꿔줘"기존 파일의 특정 코드를 변경**Edit Tool** 실행 → 문자열 치환(String Replacement)으로 `old_string`을 `new_string`으로 교체
2-2"새 파일 만들어줘"새로운 파일 생성**Write Tool** 실행 → 지정 경로에 파일 생성(File Write) 후 내용 기록
2-3"이 파일 삭제해줘"파일 제거**Bash(`rm`)** 실행 → 파일 시스템에서 해당 파일을 삭제 (확인 후 진행)
2-4"이름 바꿔줘" / "리네임해줘"파일/변수명 변경**Bash(`mv`)** 또는 **Edit Tool(`replace_all`)** → 파일명 변경 또는 코드 내 식별자(Identifier) 일괄 치환
3. 서버 / 브라우저 실행
3-1"서버 띄워줘" / "실행해줘"개발 서버를 시작하여 브라우저에서 확인**Bash** → `node server.js` 또는 `npm start` 등 서버 프로세스(Server Process)를 백그라운드(Background)로 실행
3-2"브라우저로 열어줘" / "웹에서 보여줘"HTML 파일을 웹 브라우저에서 확인**Bash** → 로컬 서버(Local Server)를 구동하거나 `npx serve` 같은 정적 파일 서버(Static File Server)를 실행
3-3"포트 바꿔줘" / "3000번으로 띄워줘"서버 포트 번호 변경**Bash** → 서버 실행 시 포트(Port) 옵션을 지정 (예: `--port 3000`)
3-4"서버 꺼줘" / "종료해줘"실행 중인 서버 프로세스 중지**Bash** → `kill` 명령어로 해당 PID(Process ID)의 프로세스를 종료
4. Git / 버전 관리
4-1"커밋해줘"변경된 내용을 Git에 저장**Bash** → `git add` (스테이징, Staging) + `git commit -m "메시지"` (커밋, Commit) 실행
4-2"푸시해줘"원격 저장소에 업로드**Bash** → `git push origin <브랜치명>` → 리모트(Remote) 저장소에 커밋 내용을 전송
4-3"브랜치 만들어줘"새로운 작업 분기 생성**Bash** → `git checkout -b <브랜치명>` → 새 브랜치(Branch) 생성 후 전환
4-4"PR 만들어줘"풀 리퀘스트 생성**Bash** → `gh pr create` → GitHub CLI를 통해 Pull Request 생성 (제목, 본문 자동 작성)
4-5"변경사항 보여줘" / "뭐가 바뀌었어?"현재 수정 내역 확인**Bash** → `git status` + `git diff` → 스테이지 상태와 변경 내용(Diff) 출력
4-6"되돌려줘" / "롤백해줘"이전 상태로 복구**Bash** → `git checkout -- <파일>` 또는 `git revert` → 변경사항을 취소 (확인 후 진행)
5. 코드 작성 / 개발
5-1"이 기능 추가해줘"새로운 기능 코드 작성관련 파일을 **Read**로 분석 → **Edit/Write Tool**로 코드 삽입 → 기존 아키텍처(Architecture)에 맞게 통합
5-2"버그 고쳐줘" / "에러 해결해줘"오류 원인 파악 및 수정에러 로그 분석 → **Grep**으로 원인 추적 → **Edit**으로 디버깅(Debugging) 및 패치(Patch) 적용
5-3"리팩토링해줘"기능은 유지하면서 코드 구조 개선코드를 읽고 중복/비효율 파악 → **Edit**으로 리팩토링(Refactoring) — 함수 분리, 변수명 개선 등
5-4"주석 달아줘" / "설명 추가해줘"코드에 설명 주석 추가**Edit Tool** → 코드 라인 위에 주석(Comment) 삽입 (`//`, `/* */`, `` 등)
5-5"타입스크립트로 바꿔줘"언어/문법 변환파일을 읽고 → **Write Tool**로 새 확장자 파일 생성 → 타입 어노테이션(Type Annotation) 추가
6. 패키지 / 의존성 관리
6-1"패키지 설치해줘" / "npm install 해줘"필요한 라이브러리 설치**Bash** → `npm install <패키지명>` → node_modules에 의존성(Dependency) 다운로드 + package.json 업데이트
6-2"빌드해줘"프로덕션용 코드 생성**Bash** → `npm run build` → 번들러(Bundler, 예: Webpack/Vite)가 코드를 압축(Minify) + 최적화
6-3"테스트 돌려줘"자동화된 테스트 실행**Bash** → `npm test` → 테스트 러너(Test Runner, 예: Jest/Vitest)가 테스트 케이스(Test Case) 실행
7. 배포 / 호스팅
7-1"배포해줘" / "디플로이해줘"웹사이트를 인터넷에 공개**Bash** → `firebase deploy` 등 → 호스팅 서비스(Hosting Service)에 빌드 결과물을 업로드
7-2"Firebase 설정해줘"Firebase 프로젝트 초기화**Bash** → `firebase init` → Firebase CLI로 프로젝트 설정 파일(`firebase.json`) 생성
8. 조사 / 분석
8-1"이 코드 분석해줘" / "설명해줘"코드의 동작 원리 파악**Read Tool**로 파일을 읽고 → 코드 로직(Logic), 데이터 흐름(Data Flow)을 분석하여 한국어로 설명
8-2"비교해줘"두 파일/방식의 차이점 비교두 파일을 **Read** → 차이점(Diff)을 분석하여 표로 정리
8-3"검색해줘" / "찾아봐줘"웹에서 정보 검색**WebSearch Tool** → 웹 검색 엔진을 통해 최신 정보를 조회하여 요약
8-4"이 URL 내용 알려줘"특정 웹페이지 내용 확인**WebFetch Tool** → 해당 URL의 HTML/텍스트를 가져와서(Fetch) 내용 요약
9. 문서 / 산출물
9-1"MD 파일로 정리해줘"마크다운 문서 생성**Write Tool** → `.md` 파일 생성, 마크다운(Markdown) 문법으로 구조화된 문서 작성
9-2"PDF로 만들어줘"PDF 파일 생성**Bash** → Puppeteer/html-pdf 등 라이브러리로 HTML을 PDF로 렌더링(Rendering) 후 파일 저장
9-3"PPT/발표자료 만들어줘"프레젠테이션 제작**Write Tool** → HTML 기반 슬라이드 파일 생성 (CSS로 레이아웃 + JS로 네비게이션)
9-4"표로 정리해줘"정보를 표 형태로 구조화텍스트 출력 또는 **Edit Tool** → 마크다운 테이블(`| 열1 | 열2 |`) 또는 HTML `` 생성
10. 프로젝트 관리 / 기타
10-1"기억해줘" / "메모해줘"다음 대화에서도 기억할 정보 저장**Write/Edit Tool** → 메모리 디렉토리(`~/.claude/.../memory/`)에 `.md` 파일로 영구 저장(Persistent Storage)
10-2"계획 세워줘" / "플랜 짜줘"작업 순서와 방향 설계**Plan Agent** 실행 → 작업을 단계별로 분해(Decomposition)하고 의존성(Dependency)을 고려한 실행 계획 수립
10-3"동시에 해줘" / "병렬로 진행해줘"여러 작업을 동시 실행**Agent Tool (병렬)** → 여러 서브에이전트(Sub-agent)를 동시에 생성하여 독립 작업을 병렬(Parallel) 처리
10-4"이전 거 취소해줘" / "원래대로 돌려줘"직전 작업 되돌리기**Edit Tool** → 변경 전 내용으로 재치환, 또는 **Bash** → `git checkout`으로 파일 복원

# 패턴 예시
A분석 → 수정"이 파일 읽고, 버그 있으면 고쳐줘"
B생성 → 실행"서버 코드 만들고 바로 띄워줘"
C수정 → 커밋"이거 수정하고 커밋까지 해줘"
D검색 → 적용"최신 방법 찾아서 적용해줘"
E전체 → 부분"전체 구조 보여주고, main.js 설명해줘"

기술 용어 뜻
File System운영체제의 파일/폴더 관리 체계
Glob Pattern`*.js`, `**/*.html` 같은 파일명 매칭 패턴
Regex (정규표현식)텍스트에서 패턴을 찾는 검색 문법
Staging (스테이징)Git에서 커밋 전 변경사항을 준비하는 단계
Commit (커밋)변경사항을 Git 히스토리에 영구 저장하는 행위
Branch (브랜치)독립적인 작업 흐름을 만드는 Git의 분기 기능
Pull Request (PR)코드 변경을 검토 요청하는 GitHub 기능
Diff두 버전 간의 차이점을 보여주는 비교 결과
Port (포트)서버가 요청을 받아들이는 네트워크 통신 번호 (예: 3000, 8080)
PID (Process ID)실행 중인 프로그램을 식별하는 고유 번호
Dependency (의존성)프로젝트가 작동하기 위해 필요한 외부 라이브러리
Bundler (번들러)여러 파일을 하나로 묶어 최적화하는 도구
Minify (압축)공백/주석을 제거해 파일 크기를 줄이는 과정
Rendering (렌더링)코드를 화면에 보이는 결과물로 변환하는 과정
Deploy (배포)완성된 코드를 서버에 올려 사용자가 접근 가능하게 하는 것
Background (백그라운드)화면 뒤에서 보이지 않게 실행되는 프로세스
Refactoring (리팩토링)기능은 그대로 두고 코드 구조/가독성만 개선하는 작업
Debugging (디버깅)코드의 오류(Bug)를 찾아 수정하는 과정
Persistent Storage프로그램 종료 후에도 데이터가 유지되는 저장 방식
