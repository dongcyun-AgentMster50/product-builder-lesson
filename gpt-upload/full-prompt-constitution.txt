- 커스텀 도시: 목록에 없으면 직접 입력 가능 → 라이브 API 호출
- 국가 선택 시 로케일 자동 전환 (COUNTRY_LOCALES 매핑)
- 지역 선택 후 라이브 인사이트 자동 수집 (기후, GDP, 도시화율, 트렌드, 이벤트)

### Q3. 타겟 생활 맥락 (Segment)
A-B-C 각 영역에서 최소 1개 선택 또는 직접 입력. MECE 구조 (21 항목):

**A. 우리 집 구성원 (Household)** — checkbox, 복수 선택 가능:
- 나 혼자
- 배우자/파트너 (하위: 맞벌이 / 외벌이)

- 부모님/시니어 (하위: 한 분 / 두 분 / 60대 / 70대 이상)
- 형제/자매
- 룸메이트/하우스메이트
- 반려동물 (하위: 강아지 / 고양이 / 기타)
- 접근성 배려 필요 구성원 (하위: 거동 불편 / 시각/청각 지원 / 인지 지원 / 영유아 안전)

**B. 요즘 관심사 (Interest)** — checkbox:
- 건강/수면/운동
- 에너지/비용 절감
- 홈시네마/음악/게임

- 외출/출장 잦음
- 홈파티/손님 맞이
- 시즌 이벤트 (명절/연말/월드컵 등)

**C. 거주지 유형 (Housing)** — radio, 국가별 로컬라이즈:
- 기본: 아파트 고층 / 아파트 저중층 / 원룸/오피스텔 / 단독주택 / 타운하우스 / 전원/교외 / 임대 / 셰어하우스

10 Life-Context Categories Framework
================================================================================

지역 트렌드를 CX 시나리오와 캠페인에 접목하기 위한 10개 생활 맥락 카테고리:

1. 기후/계절 (climate, heatwave, cold, monsoon, air quality)
   → 시나리오: 외출 전 에어컨/공기청정/블라인드 자동 제어, 수면 시간대 온도/공기질 루틴
   → 가치: Care / Save

2. 주거 형태 (housing type, apartment vs house, new developments)
   → 시나리오: 존별 냉난방 분리 제어, 현관 중심 외출/귀가 루틴
   → 가치: Ease / Secure / Save

3. 가족/돌봄 (family structure, childcare, elderly care, single households)
   → 시나리오: 아이 귀가 알림, 부모님 활동 기반 안부 케어, 가족 일정 연동
   → 가치: Care / Secure

4. 일상 리듬 (commute, remote work, daily patterns)
   → 시나리오: 출퇴근 루틴, 재택 집중 모드, 주말 청소/세탁 자동화
   → 가치: Ease / Play / Save

5. 안전/보안 (crime concerns, night safety, empty home anxiety)
   → 시나리오: 외출 시 자동 보안 모드, 여행 중 재실 시뮬레이션
   → 가치: Secure / Care

6. 에너지 비용 (electricity rates, energy policy, savings demand)
   → 시나리오: 피크 시간대 절전, 미사용 기기 자동 OFF, 월간 소비 추적
   → 가치: Save

7. 건강/웰니스 (sleep, air quality sensitivity, wellness culture)
   → 시나리오: 수면 전 조명/온도/공기질 최적화, 운동 전후 환경 조절
   → 가치: Care

8. 펫 라이프 (pet ownership, pet care while away)
   → 시나리오: 펫 단독 재실 시 온도 유지, 카메라 기반 이상 행동 알림
   → 가치: Care / Secure

9. 이동/부재 (travel, long commute, extended absence)
   → 시나리오: 외출 감지 후 절전/보안 전환, 귀가 예상 시간 맞춤 사전 냉난방
   → 가치: Secure / Save / Care

10. 문화 행사 (local festivals, holidays, seasonal events)
    → 시나리오: 홈파티 조명/TV/주방 루틴, 명절 귀성 부재 보안 루틴
    → 가치: Play / Care / Secure

**상위 분류 체계**:
- A. 환경 기반: 기후, 계절, 공기질, 에너지 비용
- B. 생활 기반: 주거 형태, 일상 리듬, 이동/부재 패턴, 가사 부담
- C. 관계 기반: 가족 구성, 자녀 돌봄, 부모 케어, 펫 케어
- D. 정서 기반: 불안/안심, 휴식/웰니스, 즐거움/엔터테인먼트, 특별한 순간
- E. 시즌 기반: 지역 행사, 명절, 휴가철, 개학/이사 시즌

**4문항 필터** (트렌드 → CX 시나리오 전환 가능성 판단):
1. 이 지역 고객이 자주 겪는 생활 순간인가?
2. 그 순간에 불편/불안/낭비/기대가 분명한가?
3. SmartThings가 2개 이상 기기/서비스를 연결해 해결할 수 있는가?
4. 캠페인 메시지로 한 문장에 압축 가능한가?

웹앱 아키텍처 (Web App Architecture)
================================================================================

**시스템 구성**:

```
[Client: Vanilla JS + HTML/CSS]
     | (HTTPS)
[Server: Node.js HTTP — server.js]
     |-- /api/access/*       → 인증 (세션 쿠키)
     |-- /api/region-insight  → Q2 라이브 지역 인사이트
     |-- /api/nudge           → 도시 넛지 카드 (Now/Need/Opportunity)
     |-- /api/generate        → 시나리오 생성 (SSE 스트리밍)
     |-- /api/refine          → 시나리오 수정/확장 (SSE 스트리밍)
           |
     [External APIs]
     |-- OpenAI (GPT-4o-mini): 라이브 트렌드, 넛지
     |-- OpenAI (GPT-5.4) / Claude (claude-sonnet-4): 시나리오 생성
     |-- World Bank: GDP, 도시화율
     |-- Open-Meteo: 지오코딩, 7일 기상
     |-- Nominatim (OpenStreetMap): 도시 메타데이터
     |-- REST Countries: 국가 프로필
```

**사용자 플로우**:
1. Access Code 입력 → 인증
2. Guide 화면 (사용법 선택)
3. Q1: 관점 선택 (Retailer / Dotcom / Brand)
4. Q2: 국가 + 도시 선택 → 라이브 인사이트 자동 수집
5. Q3: 타겟 세그먼트 구성 (A 구성원 + B 관심사 + C 주거유형)
6. Q4: 기기 선택 (프리셋 또는 개별)
7. Scenario Build → SSE 스트리밍 출력
8. Refinement: 섹션 수정, 확장, 품질 점검

**AI 모델 구성**:
| 용도 | 모델 | max_tokens |
|------|------|-----------|
| 시나리오 생성 | GPT-5.4 | 8,000 |
| 시나리오 수정 | GPT-5.4 또는 Claude Sonnet 4 | 4,000 |
| 라이브 트렌드 (Q2) | gpt-4o-mini | 2,000 |
| 넛지 카드 (Q2) | gpt-4o-mini | 1,000 |

**레퍼런스 데이터 (references/)**:
