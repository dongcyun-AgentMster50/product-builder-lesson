My GPT vs Gems

Global marketer 관점의
Scenario Self Generation Agent 비교

알고 싶은 포인트를 기준으로 다시 정리했다. 단순 기능표가 아니라 접속 경로, 보이는 UI/UX, 실사용 불편성, 그리고 글로벌 마케터가 Scenario Self Generation Agent를 각 플랫폼에서 구현했을 때의 사용성을 함께 본다.

한 줄 결론

Executive readout
**My GPT**는 “에이전트 앱처럼 보이고 동작하는 전용 도착점”에 강하다. 반면 **Gems**는 Google Docs / Drive / Workspace 안에서 바로 쓰는 “문맥 내 보조자”에 강하다.
My GPT = destination appGems = in-flow assistant

마케터 시나리오 정의

What the agent should do
  • 시장/국가별 캠페인 시나리오 자동 생성
  • 포지셔닝·메시지 아키텍처·채널별 아이디어 산출
  • 브리프/가이드/과거 성과자료를 반영한 반복 작업 단축
  • 글로벌 팀이 재사용 가능한 일관된 output 확보
채택 관점에서 보면, My GPT는 “전용 툴화”에, Gems는 “문서 작업 흐름 속 침투”에 더 유리하다.

보이는 화면과 UX 감각

같은 “맞춤형 AI”여도 체감은 다르다. My GPT는 설정형 빌더이고, Gems는 Gemini + Workspace에 녹아드는 보조자 느낌이 강하다.

My GPT UI/UX 성격

Builder-first
  • Create 탭에서 GPT Builder와 대화하며 초안을 만들고, Configure 탭에서 수동 조정하는 구조
  • 도구(예: 웹, 파일, 이미지, API actions/Apps)를 명시적으로 선택하는 “세팅형 UX”
  • conversation starters와 공개 범위 설정이 있어 “하나의 앱/에이전트”처럼 패키징하기 좋음
Builder feels like product setupApp-like launch point

Gems UI/UX 성격

Workflow-first
  • Gemini web app에서 만들지만, 실제 사용은 Gemini app / mobile / Docs side panel 등 작업 맥락 안에서 이어짐
  • Gem 생성 시 오른쪽 preview 영역에서 바로 시험 가능
  • Docs에서는 Ask Gemini → Gems 경로로 문서 편집 흐름을 끊지 않고 사용 가능
OpenAI official: builder has Create and Configure tabs, lets you add instructions, knowledge, and capabilities — Creating a GPT
Google official: create preview on the right, then save; use Gems in Docs side panel via Ask Gemini → Gems — Tips for creating custom Gems, Collaborate with Gemini in Google Docs

Usability pain points

사용상 불편성 / friction point

단기 데모보다 장기 사용성이 중요하다. 실제로 팀이 계속 쓰게 하려면 “불편한 순간”을 줄이는 쪽이 유리하다.

My GPT 쪽 불편성

What may slow marketers down
  • 컨텍스트 전환: 브리프/시트/드라이브 자료를 보다가 ChatGPT로 옮겨가는 흐름이 생기기 쉽다.
  • 설정 피로: GPT Builder + Configure + 도구/액션 + 공유 범위까지 고려하면 “잘 만든 한 개”는 쉬워도 운영은 점점 제품화 과제가 된다.
  • 퍼블리시 운영 요소: public store 수준으로 가면 Builder Profile, 도메인 검증, privacy policy 같은 항목이 따라온다.
  • 플랜 의존성: GPT 생성은 유료/조직형 플랜 중심이고, free 사용자는 생성이 아니라 사용/탐색 중심이다.

Gems 쪽 불편성

What may slow marketers down
  • 저장 실수: preview 해도 자동 저장되지 않아서 “만든 줄 알았는데 저장 안 됨”이 발생하기 쉽다.
  • 기능 제약: 현재 Gems는 Gemini Live와 함께 쓸 수 없다.
  • 공유 주의: 공유된 Gem은 열고 상호작용해야 내 Gem manager에 나타나며, 업로드한 파일과 지시사항이 보일 수 있다.
  • 관리자/권한 변수: Workspace 계정에서는 공유/Drive/side panel 사용성이 조직 설정에 따라 달라질 수 있다.

마케터 관점 해석

Plain-language takeaway
My GPT의 friction은 “도구 밖으로 나가야 하는 것”이고, Gems의 friction은 “권한/저장/제약을 이해해야 하는 것”에 가깝다. 즉, My GPT는 adoption friction, Gems는 governance friction 쪽이 더 강하다.

실무 팁

If you deploy this internally
My GPT는 런칭 가이드와 prompt starter 교육이 필요하고, Gems는 저장/공유/Drive 파일 권한 가이드를 같이 배포해야 팀 혼선을 줄일 수 있다.

Final recommendation

추천안

둘 중 하나만 고르기보다, “어디에서 쓰이게 할 것인가”를 먼저 정하는 편이 맞다.

My GPT 우선 추천

When to choose first
  • Scenario agent를 하나의 제품처럼 보이게 하고 싶을 때
  • 링크 공유·명시적 도구·배포 구조가 중요할 때
  • 향후 외부 시스템/API 연계를 더 강하게 고려할 때

Gems 우선 추천

When to choose first
  • 팀의 일상이 Google Docs / Drive / Workspace 중심일 때
  • 브리프 문서를 열어둔 채 바로 시나리오를 얻는 흐름이 중요할 때
  • 글로벌 마케팅 협업에서 문서 기반 adoption 속도가 우선일 때

현실적인 하이브리드

Best practical setup
  • My GPT를 core scenario engine으로 두고
  • Gems를 Docs/Drive drafting companion으로 둔다
  • 팀이 어느 surface에서 가장 오래 머무는지에 따라 front door를 결정한다
