# 1주차 — 환경 구축 및 계획 수립

## 과제 요구사항 정의

### 핵심 질문 3가지
- Q1. Codex / Gemini가 Claude만큼 시나리오를 잘 만드는가?
- Q2. 마케터 혼자 30분 안에 결과물을 뽑을 수 있는가?
- Q3. Enterprise 환경에서 실제 업무 투입이 가능한가?

### 성공 기준
| 항목 | 기준값 |
|---|---|
| 시나리오 품질 점수 | 4점 이상 / 5점 |
| 마케터 조작 소요시간 | 30분 이내 |
| 멀티 제약 동시 이행 | 국가 + 역할 + 가치태그 3개 |

## 테스트 케이스 (TC) 정의

### TC-01 기본 시나리오 생성
```
입력:
- 역할: 리테일
- 국가: 한국 / 서울 강남
- 가치태그: Care

기대 결과: 현지화된 시나리오 1개 출력
평가: 시나리오 자연스러움 + 지역 특성 반영 + Care 가치 포함
```

### TC-02 멀티 제약 이행
```
입력:
- 역할: 브랜드
- 국가: 미국 / 뉴욕
- 가치태그: Save + Secure (2개 동시)

기대 결과: 두 가치태그 모두 반영된 시나리오
평가: 멀티 제약 이행률 (5개 항목 중 몇 개 이행)
```

### TC-03 초개인화
```
입력: TC-01 결과 +
- 거주형태: 아파트
- 가족구성: 3인 가족
- 보유기기: 에어컨

기대 결과: 기기/거주형태 반영된 개인화 시나리오
평가: 초개인화 신호 반영 여부
```

### TC-04 채널 확장
```
입력: TC-01 결과 +
- 채널 선택: 카피라이팅 + 이메일 템플릿

기대 결과: 두 채널용 결과물 동시 출력
평가: 채널별 톤 차이 + 실사용 가능성
```

### TC-05 코드 재현
```
입력: "SmartThings 시나리오 생성 웹앱 만들어줘"
      (3단계 입력 + AI 생성 + BYOK 구조)

기대 결과: v2.0 구조 재현
평가: A1~A5 에이전트 중 구현된 수 / 5개
```
