③ 시나리오 생성

④ 결과 & 수정

11개 섹션 카드 렌더링 → Refine API로 섹션별 정밀 수정.

6/18 Part B — 한눈에 보기

GPT vs 웹앱 — 10단계 개선 요약

1
접속 및 시작GPT: ChatGPT Plus 구독 필수 ($20/월/인). 접속하면 빈 채팅창만 보임. 뭘 입력해야 할지 모르는 상태에서 시작.
웹앱: 전용 URL + 액세스 코드로 무료 접속. 접속 즉시 4단계 가이드 위자드가 표시되어 첫 사용자도 바로 시작 가능. 접근 제어 + 세션 관리로 보안 확보.
2
역할/관점 설정 (Q1)GPT: "리테일 관점에서" 텍스트 입력. 역할을 안 쓰면 AI가 범용 관점으로 생성. Retail/Dotcom/Brand 구분 약함.
웹앱: 3장 역할 카드 클릭 선택. 선택 즉시 인사이트 카드("이 역할에서는 이런 흐름이 유리합니다") 표시. 이후 모든 출력에 3 Role-Lens 자동 분기.
3
지역/시장 선택 (Q2)GPT: "한국 서울에서" 텍스트 입력. 학습 데이터 기반 추측 → 가짜 URL 생성. 어떤 도시를 넣어도 비슷한 일반론.
웹앱: 국가 드롭다운 + 도시 검색. GPT-4o 웹검색 + 7개 외부 API 병렬 호출. 실제 URL 출처 + 10개 생활 카테고리 프레임워크. 도시마다 완전히 다른 결과.
4
타겟 세그먼트 (Q3)GPT: "30대 맞벌이 대상" 자유 텍스트. 세그먼트 기준 없음. 가구 형태, 기술 수용도 등 누락 빈번. 의도(intent) 별도 안 물어봄.
웹앱: MECE 21항목 체크리스트 (생애단계·가구·라이프스타일·기술수용도). AI 자동 구성 버튼. Q3 넛지 카드로 선택 가이드. 중복/누락 원천 차단.
5
기기/제품 선택 (Q4)GPT: "냉장고, 세탁기" 텍스트 나열. 존재하지 않는 기기 입력 가능. 기기 조합 효과 예측 불가. "알아서 넣어줘" → AI 임의 선택.
웹앱: 7개 스타일별 프리셋 중복 선택 (에너지/케어/무드/시큐리티 등) + 22개 기기 직접 토글 + AI 자동 구성. 실시간 "현재 조합 요약" 패널에서 가능 범위 즉시 확인.
6
지역 데이터 수집 (내부)GPT: 외부 API 호출 없음. 학습 데이터(~2024) 기반 추측. GDP, 날씨, 인구 실시간 접근 불가. 출처 = 지어낸 URL.
웹앱: 7개 외부 API 병렬 호출 — WorldBank(GDP/도시화율), Open-Meteo(날씨/좌표), Nominatim(도시분류), RestCountries(인구), Wikipedia(이미지), GPT-4o(웹검색). 모든 데이터 검증 가능.
7
CX 시나리오 생성GPT: ~200줄 프롬프트 + 사용자 메시지 1개. 지역 데이터 없이 생성. 출력 섹션 수/순서 매번 다름. Explore 매핑 없음. Gate 검증 없음.
웹앱: 550줄 prompt.txt v8.2 + Q2 regionInsight 삽입. 11섹션 고정 스키마. Explore v2.0 12키워드 자동 매핑. Gate System v2.0 (Hard/Soft) 자동 검증. GPT-4o 또는 Claude 선택.
8
결과 수신 방식GPT: 전체 완성까지 30~90초 빈 화면 대기. 토큰 초과 시 "Continue" 클릭 필요. 결과 = 긴 마크다운 텍스트 블록.
웹앱: SSE(Server-Sent Events) 실시간 스트리밍 → 생성 즉시 한 글자씩 표시. 체감 대기 80% 감소. 토큰 제한 사전 방지. 섹션별 카드 UI로 자동 파싱.
9
수정 및 반복GPT: "02번 다시 써줘" 대화식. 대화 이력 길어지면 컨텍스트 잘림 → 이전 결과 까먹음. 전체/부분 수정 구분 없음. 품질 하락.
웹앱: POST /api/refine — 전용 수정 API. 이전 결과 + 수정 지시를 구조화된 messages[]로 전달. 해당 섹션만 타겟 업데이트. 수정 후 자동 품질 재검증(Gate).
10
결과물 활용 & 공유GPT: 채팅창 마크다운 복사 → 붙여넣기. 표/JSON 깨짐 빈번. 한국어 전용. 공유 = ChatGPT 링크 (편집 불가).
웹앱: 11섹션 구조화 카드 UI. 섹션별/전체 복사. 표·JSON 정확 렌더링. 9개 로케일 자동 전환. URL 공유 → 팀원 즉시 사용. git push = 프롬프트 중앙 관리.
7/18 STEP 1/10GPT 단계별 비교

Step 1. 접속 및 시작

기존 GPT

  • ChatGPT 접속 → "나의 GPT" 검색/클릭
  • ChatGPT Plus 구독 필수 ($20/월/인)
  • 접근 제어 없음 — 링크만 있으면 누구나
  • 시작 시 아무 가이드 없음 — 빈 채팅창
비구독자 사용 불가. 처음 접속한 마케터는 뭘 입력해야 할지 모름.

웹앱

  • 전용 URL 접속 → 액세스 코드 입력
  • Cloudflare Pages 무료 호스팅 — 구독 불필요
  • 액세스 코드 기반 접근 제어 + 세션 관리
  • 접속 즉시 단계별 가이드 위자드 표시
구독 없이 누구나 사용. 접근 제어로 보안 확보. 가이드로 첫 사용 진입장벽 제거.
8/18 STEP 2/10GPT 단계별 비교

Step 2. 역할/관점 설정 (Q1)

기존 GPT

  • "리테일 담당자 관점에서 시나리오 만들어줘"
  • 역할을 안 쓰면 → AI가 범용 관점으로 생성
  • 역할별 차이가 불분명 (retail/dotcom/brand 구분 약함)
  • 역할이 시나리오 구조에 미치는 영향 불투명
역할 미입력 시 초점 없는 범용 시나리오. 3가지 역할의 차이를 사용자가 직접 이해해야 함.

웹앱 Q1

  • 3장의 역할 카드 중 클릭으로 선택
  • 각 카드에 역할 설명 + 포커스 영역 명시
  • 선택 즉시 Q1 인사이트 카드 표시 — "이 역할에서는 이런 흐름이 유리합니다"
  • 역할이 이후 모든 출력(메시지 3 Role-Lens, KPI, 캠페인)에 자동 반영
역할 누락 0%. 선택 즉시 해당 관점의 시나리오 구조가 결정됨. 3 Role-Lens 자동 분기.
9/18 STEP 3/10GPT 단계별 비교

Step 3. 지역/시장 선택 (Q2)

기존 GPT

  • "한국 서울에서" 텍스트로 입력
  • 국가/도시 스펠링 오류 → AI 오해석
  • 지역 데이터 = AI 학습 데이터에서 추측
  • 출처 URL = 가짜 (hallucination)
  • 어떤 도시를 넣어도 비슷한 결과
  • 행사/날씨: 과거 데이터 혼입
가짜 출처. 2023년 데이터. 도시별 차이 없음. 클릭해도 "페이지 없음".

웹앱 Q2

  • 국가 드롭다운 (9개 로케일) + 도시 검색/자동완성
  • 선택 즉시 라이브 트렌드 카드 자동 생성
  • GPT-4o + web_search_preview = 실제 웹 검색
  • 7개 외부 API 병렬 (GDP, 날씨, 좌표, 인구)
  • 10개 생활 카테고리 프레임워크
  • 과거 이벤트 서버 필터링 (todayIso 이후만)
  • 출처: 실제 URL + [Source] 표기
구 단위 구체적 트렌드. 진짜 URL. 최신 데이터만. 도시마다 완전히 다른 결과.
10/18 STEP 4/10GPT 단계별 비교

Step 4. 타겟 세그먼트 설정 (Q3)

기존 GPT

  • "30대 맞벌이 가구 대상으로" 텍스트 입력
  • 세그먼트 분류 기준 없음 — 사용자 자의적
  • 가구 형태, 라이프스타일, 기술 수용도 등 누락 빈번
  • 사용 의도(intent)를 별도로 안 물어봄
세그먼트 정의가 모호하거나 불완전. AI가 추측으로 보충하여 의도와 다른 시나리오 생성.

웹앱 Q3

  • MECE 21항목 체크리스트 (생애 단계, 가구, 라이프스타일, 기술 수용도 등)
  • radio (단일 선택) + checkbox (복수 선택) 혼합
  • 사용 의도 텍스트 입력란 별도 제공
  • Q3 넛지 카드 (/api/nudge) — AI가 세그먼트 선택 가이드
  • 선택 즉시 Q3 인사이트 카드 → 시나리오 방향 힌트
세그먼트 정의 완전성 보장. MECE라 중복/누락 없음. 넛지가 선택을 안내.

Step 5. 기기/제품 선택 (Q4)

기존 GPT

  • "냉장고, 세탁기, 에어컨" 텍스트로 나열
  • SmartThings 생태계 기기 목록을 사용자가 외워야 함
  • 기기 조합의 시나리오 영향 불투명
  • "알아서 적절히 넣어줘" → AI 임의 선택
존재하지 않는 기기 입력 가능. 조합 효과 예측 불가. 결과의 기기 일관성 없음.

  • 실시간 "현재 조합 요약" — 가능한 기능 + 제한 표시
  • 카테고리별 상세 보기 (접이식)
기기 조합의 시나리오 효과 즉시 확인. 프리셋으로 3초 선택. AI 자동 구성으로 완전 위임 가능.

  • 출처 = 지어낸 URL

7개 실시간 소스. 공식 통계 + 웹 검색 + 날씨. 모든 데이터 검증 가능.

Step 8. 결과 수신 방식

기존 GPT

  • 한 글자씩 채팅창에 표시 (ChatGPT 자체 스트리밍)
  • 전체 완성까지 30~90초
  • 중간에 토큰 초과 → "Continue generating" 클릭
  • 결과 = 마크다운 텍스트 블록
  • 섹션 구분 불명확 — 스크롤해서 찾기
긴 대기. 토큰 잘림. 결과물이 긴 텍스트 블록이라 섹션별 활용 어려움.

웹앱

  • SSE(Server-Sent Events) 실시간 스트리밍
  • 생성 즉시 한 글자씩 UI에 표시
  • 토큰 제한 → max_tokens 설정으로 사전 방지
  • 섹션별 카드 UI로 자동 파싱
  • 진행률 표시 (Q1 → Q2 → Q3 → Q4 → 생성 중)
  • 생성 완료 후 섹션별 탭/접기 지원
체감 대기 80% 감소. 토큰 잘림 없음. 섹션별 카드로 즉시 활용 가능.

Step 9. 수정 및 반복

기존 GPT

  • "위에서 02번 상세 시나리오만 다시 써줘" 대화식 요청
  • 대화 이력이 길어지면 → 컨텍스트 잘림
  • 이전 결과를 AI가 까먹음 → 일관성 하락
  • 전체 재생성 vs 부분 수정 구분 없음
  • 여러 번 수정 → 대화창이 매우 길어짐
대화 길어질수록 품질 하락. 이전 맥락 유실. 부분 수정 불가.

웹앱

  • POST /api/refine — 전용 수정 API
  • 이전 결과 + 수정 지시를 구조화된 messages[]로 전달
  • 해당 섹션만 타겟 업데이트 (전체 재생성 불필요)
  • 수정 후 자동 Reflection Check (11번) 재실행
  • Q1~Q4 변경 시 → 즉시 영향받는 섹션만 재생성 제안
부분 수정 → 속도/비용 절감. 대화 이력 구조화로 맥락 유실 없음. 수정 후 품질 재검증.

Step 10. 결과물 활용 & 공유

  • 표/JSON이 깨지는 경우 빈번
  • 공유: ChatGPT 대화 공유 링크 (편집 불가)
  • 다국어: 한국어로 질문 → 한국어 결과만
  • 이력 관리: ChatGPT 대화 목록에 묻힘
수동 복사붙여넣기. 포맷 깨짐. 다국어 불가. 결과물 재활용 어려움.

  • 섹션별 복사 / 전체 내보내기
  • 표/JSON 구조 정확히 렌더링
  • 9개 로케일 자동 전환 (한/영/독/불/서/포/이/네/아랍)

  • 프롬프트 중앙 관리: git push = 전체 반영
깔끔한 보고서 포맷. 9개 언어 즉시 전환. 팀 공유 원클릭. 프롬프트 버전 중앙 관리.

근본적 아키텍처 차이: 대화형 vs 파이프라인형

GPT: 대화형 (Conversational)

사용자 → [자유 텍스트]
         ↓
    ChatGPT (단일 LLM)
         ↓
    [마크다운 텍스트]
         ↓
       사용자

단일 모델, 단일 호출, 단일 출력. 모든 것이 LLM의 상상에 의존.

웹앱: 파이프라인형 (Orchestrated)

사용자 → [구조화 4단계 입력]
         ↓
    ┌─ WorldBank API ──┐
    ├─ OpenMeteo API ───┤
    ├─ Nominatim API ──┤ 병렬
    ├─ RestCountries ──┤
    ├─ Wikipedia API ──┤
    └─ GPT-4o + Web ───┘
         ↓
    prompt.txt v8.2 + regionInsight
         ↓
    GPT-4o / Claude (SSE)
         ↓
    [11섹션 구조화 보고서]

다중 모델, 다중 소스, 구조화 출력. 실시간 데이터 + AI 생성 + 검증 파이프라인.
