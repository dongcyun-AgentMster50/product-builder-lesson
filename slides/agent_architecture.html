설계 목적

에이전트의 존재 이유와 통제 목표

이 에이전트는 단순한 채팅봇이 아닙니다. 정확한 데이터를 기반으로 마케터에게 확실한 마케팅 시나리오를 산출해 내는 정교한 '소프트웨어'입니다.

무엇을 위한 부분인가?

Target Objective
마케터가 거대한 프롬프트 지시사항을 학습할 필요 없이, 간단한 환경(국가, 페르소나, 기기) 입력만으로 현지화된 캠페인 전략, 카피, 스토리보드를 추출할 수 있도록 돕는 것을 목표로 합니다.

결과 및 활용

Result & Usage
AI의 무분별한 상상력을 통제하고, DB에 기록된 공식적인 삼성 기기 지원 여부와 핵심 가치를 보존합니다. 이를 통해 마케터는 에이전트를 브랜드 통제력을 잃지 않는 검증된 툴킷으로 활용할 수 있습니다.

2. Tech Stack
3 / 10
개발 환경 구성

가벼우면서도 강력한 아키텍처 스택 (Frontend & Backend)

React 등 무거운 프레임워크를 걷어내고 바닐라(Vanilla) 기술을 채택해 극강의 퍼포먼스를 뽑아냅니다.

어떤 스택이 쓰였나?

Technologies
  • Frontend: 바닐라 JS(ES6+) + 순수 HTML/CSS 기반의 SPA 구성.
  • Backend: Node.js를 이용한 기본 `http` 프로세스. 정적 파일 호스팅과 스트리밍(SSE) 게이트웨이를 동시 처리.

이렇게 선택한 이유와 결과

Why & Outcome
의존성 패키지(Node Modules)가 꼬이는 문제를 최소화하고, 서버 비용을 획기적으로 낮췄습니다. 어떠한 기기나 환경에서도 딜레이 없이 마법사 UI 애니메이션과 실시간 AI 로딩이 동작하도록 만듭니다.

단일 AI 모델에 의존하지 않고, 작업의 특성(속도 vs 창의력)에 맞춰 최적의 모델을 골라 사용합니다.

모델별 역할 배분

GPT & Claude
  • GPT-4o Mini: 가장 응답 속도가 빠른 모델로, `Nudge Card(인사이트 팝업)` 생성처럼 사용자의 클릭 직후 2초 내에 응답이 필요한 곳에 투입됩니다.
  • GPT-4o / Claude Sonnet: 깊은 마케팅적 은유 발상과 카피라이팅이 필요한 최종 13섹션 Output 파이프라인에서 정밀도를 높이는 데 사용됩니다.

결과 및 활용

Cost & Performance
결과적으로 토큰 처리 비용(API Cost)을 크게 낮추면서도 사용자 체감 대기시간은 줄이고, Output의 창의성과 품질은 높은 수준으로 방어할 수 있게 되었습니다.

사용자 접점

프론트엔드 UI: 4단계 시각 마법사 (Wizard Flow)

채팅으로 타이핑하던 방식을 '선택형' 클릭 마법사로 교체하여 휴먼 에러를 방지합니다.

Step무엇을 위한 구성인가?활용 효과
Q1 (Role) 리테일/닷컴/브랜드 등 마케터의 담당 업무를 드롭다운으로 선택 결과 출력 시, 해당 시나리오를 각자의 KPI(방문수 vs 인지도 등)에 맞춰 변환 작성토록 도움.
Q2 (Location) 타겟 국가와 도시를 콤보박스에서 정교하게 선택 /api/region-insight 를 호출, 현지 기후와 인프라 데이터를 뒤에서 은밀히 취합함.
Q3 (Persona) 주거형태/라이프스타일/세대구성을 체크박스로 시각화 오타가 생길 수 없는 직관적 체크를 통해, 확실한 매칭 가중치(Tag Point) 부여.
Q4 (Devices) SmartThings 연결 가능한 제품들을 이미지 그리드로 아이콘화 사용자가 어떤 기기를 지원하는지 몰라도 편하게 클릭으로 선택 가능.

5. Locale Validation
6 / 10
동적 카탈로그

로케일(Locale) 동적 매핑 엔진

사용자가 특정 국가(예: 한국, 미국, 호주)를 고르는 즉시 제품 판매 리스트가 맞춰서 조율됩니다.

동적 카탈로그의 원리

LOCALE_CATALOGS
`main.js`의 `resolveLocaleCatalog` 함수를 통해, Q2에서 선택한 국가 코드를 감지하고 해당 국가에서 판매되지 않는 기기 목록을 화면의 Q4(기기 선택 장)에서 아예 삭제/비활성화 해버립니다.

결과 및 전략적 활용

Zero Compatibility issues
영국(UK) 마케터가 실수로 자국에서 팔지도 않는 스마트 오븐을 선택하는 불상사가 사라집니다. 각 글로벌 법인 마케터들은 본사에 질문할 필요 없이 자신들의 지역에 100% 호환되는 장비만으로 시나리오를 짤 수 있게 됩니다.

알고리즘의 심장

큐레이션 엔진 파트 1: Tag 빌딩

AI의 이해를 돕기 위해, 사용자의 UI 선택값을 시스템 내부의 정형화된 Tag 배열로 통일시킵니다.

태그 빌딩 프로세스

buildExploreTagsFromInput
`curation-engine.js` 에 소속된 이 함수는 Q3(1인가구, 애완동물)와 Q4(로봇청소기, 펫드라이어) 항목들을 읽고, `["Care for pets", "Clean home"]` 과 같이 Explore DB에 정의된 "Value Tag" 이름으로 일괄 치환해주는 역할을 합니다.

왜 이 과정이 필요한가?

Normalizing Intent
UI에서 마케터가 누른 버튼과 DB가 아는 용어가 다를 수 있습니다. 이를 중간에서 통역해주는 수학적 교환대입니다. 결과적으로 모델과 디바이스 간의 관계 논리를 매우 깔끔하게 연결시킵니다.

가중치 매칭 교집합

큐레이션 엔진 파트 2: 스코어링(Scoring)

변환된 Tag들을 DB상의 수백 개 오리지널 시나리오와 대조하여 가장 점수가 높은 것을 찾아냅니다.

스코어링 수식 원리

scoreScenario
`explore_db.json` 파일을 돌면서 각 시나리오가 보유한 태그와, 사용자가 입력한 태그의 "교집합" 개수를 셉니다. 여기서 사용자가 입력한 핵심 기기(예: TV)가 겹치면 특별 보너스 점수(+5점)를 부여하는 가중치 룰이 구동됩니다.

어떤 결과가 나오는가?

Ranked Results
상대평가를 통해 1위부터 5위까지의 최적합 시나리오가 "확정" 됩니다. 이는 10번을 눌러도 동일 조건이면 무조건 동일한 1위 시나리오가 떨어지는 결정론적 알고리즘을 성립시킵니다.

캠페인 렌더링 시작

캠페인 출력 엔진 (01~04 로컬 렌더링)

13섹션 중 초반부 서류는 AI 호출의 대기시간 없이 0초 만에 화면에 그립니다.

로컬 렌더링이란?

Zero API Cost
Sections 01~04 (입력정보, Top 매칭 결과, 왜 추천되었는지, 타 시나리오 비교) 부분은 JS 알고리즘의 결론값을 그대로 가져와 HTML Card 형태로 프론트엔드에서 즉석 생성합니다. API를 타지 않습니다.

어떻게 브리핑에 쓰이는가?

Transparency UX
마케터가 "이 결과가 왜 도출되었는지" 궁금할 때, 03 단락에서 "A 기기와 B 페르소나 가치가 겹쳐 점수가 높았다"라는 수학적 증거를 보여줍니다. 내부 컴플라이언스와 논리적 승인 라인을 통과하는 데 큰 무기가 됩니다.

9. Parallel API Stream
10 / 10
분할 동시 생성

병렬 스트리밍 엔진 (05~12 AI 생성) & 종합

가장 창의적인 영역(글쓰기 묘사)만을 골라 8개의 프로세스로 쪼개어 동시에 전송합니다.

동시(Promise.all) 스트리밍

Parallel Load
`campaign-output.js`에서 각 단락별(카피, 스토리보드, FGD)로 전문화된 소형 프롬프트로 나눈 뒤, Server-Sent Events(SSE)로 백엔드에 동시에 8개의 비동기 발송을 보냅니다.

결과 혁신과 활용법

Ultra-low latency
결과적으로 13단락짜리 방대한 기획서를 대기하는 데 단 몇 초 밖에 걸리지 않게 됩니다. 마케터는 타자를 치듯 실시간으로 그려지는 카드를 보며, 바로 다음 작업을 기획하거나 영감을 실시간으로 얻게 됩니다.
이 강력하고 체계적인 아키텍처는 마케터의 비효율적인 대기 시간을 증발시키고 오직 안정적 Output을 도출하는 도구로 자리매김합니다.
